# 브런치 마이그레이션 가이드

## 작업 개요
main 브런치의 모든 파일을 삭제하고 naver-realty 브런치를 기본(default) 브런치로 설정합니다.

## 현재 상태
- **main 브런치**: LICENSE와 README.md 파일만 포함
- **naver-realty 브런치**: 전체 애플리케이션 코드 포함 (13개 파일)

## 작업 완료 방법

### 방법 1: 자동 스크립트 실행 (권장)

```bash
./migrate-branches.sh
```

이 스크립트는:
1. main 브런치의 파일 삭제를 푸시합니다
2. GitHub CLI(gh)가 설치되어 있으면 자동으로 기본 브런치를 변경합니다

### 방법 2: 수동 실행

#### 2-1. main 브런치 파일 삭제

```bash
# main 브런치로 체크아웃
git checkout main

# 변경사항 푸시 (이미 로컬에 커밋되어 있음)
git push origin main
```

**참고**: 파일 삭제는 이미 로컬에서 커밋되었습니다 (커밋 해시: a4e3bef)

#### 2-2. 기본 브런치를 naver-realty로 변경

**GitHub CLI 사용:**
```bash
gh repo edit Mosframe/realty --default-branch naver-realty
```

**웹 인터페이스 사용:**
1. https://github.com/Mosframe/realty/settings/branches 접속
2. "Default branch" 섹션에서 현재 기본 브런치 확인
3. 브런치 이름 옆의 전환 아이콘(연필 모양) 클릭
4. 드롭다운에서 "naver-realty" 선택
5. "Update" 클릭
6. 변경사항 확인

## 작업 검증

작업 완료 후 다음을 확인하세요:

1. **main 브런치 확인**
   ```bash
   git checkout main
   ls
   # 아무 파일도 보이지 않아야 함
   ```

2. **기본 브런치 확인**
   - 저장소 홈페이지에서 "naver-realty"가 기본 브런치로 표시되어야 함
   - 새로 클론하면 자동으로 naver-realty 브런치를 사용함

3. **저장소 페이지 확인**
   ```bash
   # 또는 GitHub에서 확인
   # https://github.com/Mosframe/realty
   ```

## 왜 이런 방식인가?

자동화 시스템에는 다음과 같은 제한사항이 있습니다:
- 보호된 브런치(main 등)에 직접 푸시할 수 없음
- 저장소 설정(기본 브런치 등)을 변경할 수 없음
- Pull Request 브런치 컨텍스트 내에서만 작업 가능

따라서 저장소 수준의 변경을 위해서는 적절한 권한을 가진 사용자의 수동 개입이 필요합니다.

## 문제 해결

### Push 권한 에러
만약 푸시 중 권한 에러가 발생하면:
1. GitHub 계정에 저장소 쓰기 권한이 있는지 확인
2. 인증이 올바르게 설정되어 있는지 확인 (SSH 키 또는 Personal Access Token)

### GitHub CLI 인증 에러
```bash
# GitHub CLI 로그인
gh auth login
```
